# V31 Voice — стартовый док

Доступный голосовой компаньон для незрячих барабанщиков. Управление барабанным модулем **Roland V31** (киты TD-313/316/VAD316) через телефон: озвучка состояния, редактирование настроек, сборка кастомных китов — без необходимости видеть экран модуля.

**Статус:** старт. Этот док — отправная точка для работы в Claude Code.

---

## Проблема

Незрячий барабанщик играет на Roland V31. Модуль — цветной экран без какой-либо accessibility: выбор кита, настройки, редактирование звука требуют зрения. Вендор ничего для слепых не дал.

## Ключевое архитектурное решение

**Не зеркалить экран модуля — заменить его.** Состояние UI/курсора/меню модуль по MIDI не отдаёт (только модель данных). Поэтому экран модуля копировать бессмысленно. Вместо этого телефон становится доступным интерфейсом (screen reader + TTS), а модуль — звуковым движком и физической поверхностью. Общение с модулем — через MIDI SysEx, полностью задокументированный в **V31 MIDI Implementation v2.00**.

## Платформы и архитектура

Кросс-платформа **iOS + Android**, единый core.

- **Core — Rust, platform-agnostic, без I/O.** Чистая логика:
  - SysEx codec: RQ1/RQ-ответы, DT1, контрольная сумма, арифметика 4-байтных 7-битных адресов, упаковка/распаковка ниблов.
  - Карта адресов и типизированная модель параметров; конвертация значение ↔ человекочитаемая строка (для озвучки скринридером).
  - Модель кита, edit-команды, конечный автомат **write → readback → verify**.
  - Экспорт через **UniFFI** (приоритет) → биндинги Swift/Kotlin. Альтернатива — ручной C ABI + cbindgen.
- **Платформенные слои — тонкие, нативные:**
  - **MIDI-транспорт:** Android `android.media.midi`, iOS `CoreMIDI`. USB-C MIDI — основной канал, BLE-MIDI — вторичный. Слой только перекладывает байты ↔ Rust core.
  - **Анонсы:** канал скринридера — Android `announceForAccessibility`, iOS `UIAccessibility.post(.announcement)`. Своего TTS нет: голос — это скринридер, который включает сам пользователь ([ADR-0014](adr/0014-screen-reader-is-the-only-voice.md)).
  - **UI:** нативный, accessibility-first, дружит с TalkBack / VoiceOver.
- **Данные:** карта параметров + каталог инструментов (распарсить из **V31 Data List**), встроены как JSON.

## Протокол (подтверждено по спеке)

- Roland SysEx, Model ID `01 06 01`, Device ID по умолчанию `10h`.
- **Чтение (RQ1):** `F0 41 dev 01 06 01 11 <addr×4> <size×4> sum F7`
- **Запись/ответ (DT1):** `F0 41 dev 01 06 01 12 <addr×4> <data…> sum F7`
- **Checksum** = `(128 − (Σ(addr+data) mod 128)) mod 128`. Golden-векторы для тестов — в спеке §4 (примеры 1 и 2), забить как unit-тесты core.
- Адрес — 4 байта, по 7 бит каждый.
- **Текущий кит:** addr `00 00 00 00`, size 4, KitNum 0–199 (упакован ниблами).
- **Кит k:** база `04 00 00 00`, шаг `00 04 00 00` на кит (кит 200 = `0A 1C 00 00`).
  - Имя кита: `[KitCommon]` offset 0, 16 ASCII-байт. Темп: offset `00 6D`.
- **События в реальном времени:**
  - Program Change при смене кита — ненадёжен (≤128, зависит от маппинга PROG CHG) → как primary не использовать, **поллить `[Current]`**.
  - **Transmit Edit Data = ON** → модуль сам пушит DT1 при кручении на железке → парсим адрес+значение → озвучиваем.

## Скоуп фич

**MVP (сначала валидируем):**
- Подключение (USB-C MIDI), Identity-хендшейк.
- Озвучка текущего кита (номер + имя) и темпа, live при смене (полл Current).
- Переключение кита из доступного списка (PC или DT1 в Current).
- **Закрыть открытый вопрос про сохранение** (см. риски).

**V1:**
- Редактор глобальных настроек (`[Setup]`: выходы, метроном, мастер, EQ).
- Редактор триггеров/чувствительности по пэдам (`[Trigger]`).
- Полный редактор кита: переименование, выбор инструмента по пэду/слою (нужен каталог из Data List), pitch/decay/transient, громкость/пан, pad EQ/comp, сенды.
- FX: выбор типа эффекта (94 именованных) + пресеты — **не** сырые 32 параметра.
- Эмбиенс: room/overhead/reverb/resonance — уровни и типы.
- Live-озвучка правок через Transmit Edit Data.
- Каждая правка: **write → readback → озвучить фактическое значение** (для слепого слепая запись недопустима).

**Вне скоупа / невозможно:**
- Зеркалирование текста экрана, состояния курсора и навигации по меню (нет в MIDI-модели).
- Глубокая покрутка отдельных FX-параметров вслепую (generic `−20000..20000`, смысл зависит от типа эффекта — непрозрачно).

## Риски и открытые вопросы

- **Персистентность:** правка живёт во временном буфере или пишется в слот? Нужна ли отдельная команда WRITE? → проверить на железе в PoC.
- **Каталог инструментов:** номер ↔ имя из Data List; список расширяется экспаншенами Roland Cloud → не статичен.
- **BLE-MIDI** — разрешения и стабильность хуже USB-C → USB-C primary.
- **Биндинг Rust↔мобилки:** UniFFI vs ручной C ABI.
- **TTS:** латентность и очередь, чтобы реплики не наслаивались при игре.

## Следующие шаги
